Module 1 — Web Stack Fundamentals

Week 1
Outcome: Describe the components of a scalable, distributed web stack.
Key Ideas
  • Front End (client): HTML, CSS, JavaScript in browser/mobile apps.
  • Back End (server): Web servers, databases, OS, application servers, load balancers.
  • Scalability: Architectures evolve with traffic growth (horizontal/vertical scaling).
  • Distribution: Components split across tiers and nodes for reliability and performance.
Stack Snapshot
  • Browsers: Chrome, Edge, Safari, etc.
  • Web servers: Apache, IIS, Nginx, Express (Node.js).
  • Application servers: JBoss, GlassFish, WebLogic, WebSphere.
  • Load balancers: HAProxy, Nginx, AWS ELB (software/hardware).
  • Operating systems: Linux/Unix, Windows, macOS.

Module 2 — Web Servers Part 1: Apache & Nginx

Week 2
Outcome: Install and manage Apache HTTP Server and Nginx.
Apache Highlights
  • Open-source, modular, long-standing ecosystem and community.
  • Beginner-friendly configuration; frequent security updates.
  • May underperform at extreme concurrency without tuning.
Nginx Highlights
  • Event-driven architecture; predictable performance under high load.
  • Doubles as reverse proxy, load balancer, and HTTP cache.
  • Handles static content efficiently; common in modern stacks.
Ops Notes
  • Key config: nginx.conf (locations vary by OS).
  • Lifecycle: nginx -s reload|stop|quit|reopen; systemctl/service helpers on Linux.

Module 2 — Web Servers Part 2: IIS & Express.js

Week 3
Outcome: Describe IIS, Node.js, and Express.js.
IIS (Microsoft)
  • Supports HTTP/2, HTTPS, FTP/FTPS, SMTP, NNTP; Windows-centric.
  • IIS Express is a lightweight developer edition.
  • Terminology differs from Apache (DocumentRoot vs Site Home, etc.).
Node.js & Express
  • Node.js: async, event-driven runtime for scalable network apps.
  • Express: minimalist web framework for APIs and apps.
  • Stacks: MEAN/MERN (Node + Express) and LEMP (with Nginx).
Lab Focus
  • Install and manage IIS; compare with Node/Express dev flow.

Module 3 — Deployment & Hosting

Week 4
Outcomes: Deploy web servers; move apps from local to remote hosts.
Process & Pipelines
  • SDLC models: Waterfall vs Agile (shorter cycles ≠ “just mini-waterfall”).
  • CI/CD basics: integrate early, deploy often, automate checks.
Hosting Options
  • Static: GitHub Pages.
  • Dynamic: Heroku (plus AWS, Azure, others).
  • Environment parity matters from dev → staging → prod.

Module 4 — Security Part 1: SSL/TLS

Week 5
Outcome: Configure TLS & X.509; generate a CSR with OpenSSL.
Core Concepts
  • TLS secures transport (HTTPS default port 443); SSL is legacy.
  • X.509 defines certificate format for public-key infrastructure.
  • TLS handshake negotiates ciphers, verifies identity, derives session keys.
Hands-On
  • OpenSSL: key generation → CSR → obtain certificate → configure server.

Module 4 — Security Part 2: PKI & Reverse Proxy

Week 6
Outcomes: Explain PKI; integrate apps with reverse proxies/application servers.
PKI Essentials
  • Symmetric vs Asymmetric crypto; keys are related and purpose-specific.
  • Components: public/private keys, session keys, CA & intermediate CAs, cert chains.
Reverse Proxy Integration
  • Reverse proxy terminates TLS, forwards to upstreams, adds caching and routing.
  • Common tools: Nginx, HAProxy; supports header auth, rate limiting, blue/green.

Module 4 — Security Part 3: OAuth

Week 7
Outcome: Implement token-based authentication mechanisms including OAuth.
AuthN vs AuthZ
  • Authentication (AuthN): verify identity.
  • Authorization (AuthZ): verify permissions/scopes.
OAuth Overview
  • Open standard for access delegation — share data without sharing passwords.
  • Used by major providers (Google, Microsoft, Facebook, etc.).
  • Relies on tokens and scopes; integrates well with APIs and mobile/web apps.

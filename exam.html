1. Web Stack Basics

1.1 Eight Components of a Web Stack

Know the labels and examples.

  • OS (Operating System) – Base platform (e.g., Linux, Windows Server).
  • Web Server – Handles HTTP/HTTPS requests (Apache, Nginx, IIS).
  • Runtime / Language – Executes backend code (Node.js, PHP, Python, Java).
  • Framework – Structure & libraries (Express, Django, Laravel, Spring).
  • Application Code – Business logic, routes, controllers.
  • Database (DB) – Stores persistent data (MySQL, PostgreSQL, MongoDB).
  • Cache – Faster access to repeated data (Redis, Memcached).
  • Load Balancer / Reverse Proxy – Distributes traffic across servers.

1.2 Frontend vs Backend

  • Frontend – Code running in the browser (HTML, CSS, JS, React, Angular).
  • Backend – Code running on the server (Node, PHP, Python, DB operations).

1.3 Client-side vs Server-side

  • Client-side – Executed in the browser; affects UI and interactivity.
  • Server-side – Executed on server; handles logic, security, database access.

1.4 Common Stack Acronyms

  • LAMP – Linux, Apache, MySQL, PHP.
  • WAMP – Windows, Apache, MySQL, PHP.
  • MAMP – macOS, Apache, MySQL, PHP.
  • LEMP – Linux, (Engine X) Nginx, MySQL, PHP.
  • MEAN – MongoDB, Express, Angular, Node.js.
  • MERN – MongoDB, Express, React, Node.js.

2. DNS & Networking Concepts

2.1 DNS

  • DNS – Domain Name System; converts domain names (e.g., example.com) into IP addresses.

2.2 Proxy vs Reverse Proxy

  • Proxy – Sits between client and internet; hides the client (e.g., corporate proxy).
  • Reverse Proxy – Sits in front of servers; hides the servers (e.g., Nginx load balancer).

2.3 Single Point of Failure (SPOF)

  • SPOF / SPoF – Single Point of Failure; one component whose failure breaks the whole system.
  • Solution – Add redundancy: multiple servers, replica databases, extra load balancers.

2.4 C10K Problem

  • C10K – The challenge of handling 10,000 concurrent client connections efficiently.

3. Web Servers & Hosting Concepts

3.1 What is a Web Server?

  • Software that listens on ports (80/443) and responds to HTTP/HTTPS requests.
  • Serves static files or forwards requests to backend code.

3.2 Popular Web Servers

  • Apache HTTP Server – Classic, module-based, uses .htaccess and virtual hosts.
  • Nginx – High-performance, event-driven, often used as reverse proxy/load balancer.
  • IIS – Internet Information Services, Microsoft’s Windows web server for ASP.NET apps.

3.3 HTTP / HTTPS Bindings

  • Binding = IP Address + Port + Host Name (and certificate for HTTPS).
  • HTTP – HyperText Transfer Protocol (usually port 80).
  • HTTPS – HyperText Transfer Protocol Secure (HTTP over TLS, usually port 443).

3.4 IIS Concepts

  • Application Pool – Worker process group; isolates and manages web apps on IIS.
  • Feature Delegation – Controls which settings can be changed in web.config by site owners.
  • Virtual Host (VHost) – Hosting multiple domains on a single web server instance.

4. Security, Certificates & Auth

4.1 SSL, TLS & HTTPS

  • TLS – Transport Layer Security; modern protocol securing data in transit.
  • SSL – Secure Sockets Layer; older version, now deprecated but still used as a generic term.
  • HTTPS = HTTP + TLS/SSL encryption.

4.2 PKI & Certificates

  • PKI – Public Key Infrastructure; system that manages keys and certificates.
  • CA – Certificate Authority; trusted organization that issues digital certificates.
  • PEM – Privacy Enhanced Mail; Base64-encoded text format for keys and certificates.

4.3 Certificate Requests & Verification (OpenSSL)

  • Generate CSR (Certificate Signing Request):
    openssl req -new -newkey rsa:2048 -nodes -keyout site.key -out site.csr
  • Verify certificate:
    openssl x509 -in cert.crt -text -noout and openssl verify cert.crt

4.4 AuthN, AuthZ & OAuth

  • AuthN – Authentication; verifying who the user is (identity).
  • AuthZ – Authorization; verifying what the user is allowed to do.
  • OAuth – Open Authorization; protocol for delegated authorization (e.g., “Login with Google”).

5. SDLC, Git, Release & CI/CD

5.1 SDLC & Deployment

  • SDLC – Software Development Life Cycle: planning → design → coding → testing → deployment → maintenance.
  • Deployment – Moving a built application into a live/production environment.

5.2 Git & GitHub Pages

  • Git – Version control; commands like git add, git commit, git push, git pull.
  • GitHub Pages – Static site hosting directly from a GitHub repository.

5.3 Release Cycle

  • Typical order: Alpha → Beta → Release Candidate (RC) → Stable.

5.4 CI / CD

  • CI – Continuous Integration; automatically builds/tests when code is merged.
  • CD – Continuous Delivery / Continuous Deployment; automatically delivers or deploys new versions.

6. Load Balancing, CDN, Performance & Availability

6.1 Load Balancer & Algorithms

  • Load Balancer (LB) – Distributes incoming traffic across multiple servers.
  • LB Algorithms – Round-robin, least connections, IP hash, weighted round-robin.

6.2 Scalability & Distribution

  • Scalability – Ability to handle more load (vertical: bigger server; horizontal: more servers).
  • Distributed Stack – Components spread across multiple servers/regions, often with replication.

6.3 CDN

  • CDN – Content Delivery Network; caches static content on edge servers worldwide to reduce latency.

6.4 Performance & Availability

  • Performance – Measured by response time, page load time, throughput (requests per second).
  • Availability – Measured by uptime %, MTBF (Mean Time Between Failures), MTTR (Mean Time To Repair).

6.5 Static vs Dynamic Content

  • Static Content – Prebuilt files (HTML, CSS, JS, images); same for every user.
  • Dynamic Content – Generated at request time (e.g., PHP/Node returning DB-driven pages).

7. Logs & Centralized Logging

7.1 Logs & Log Levels

  • Log – Recorded events from applications or servers; used for debugging, security, and auditing.
  • Common log levels: DEBUG, INFO, WARN, ERROR, FATAL.

7.2 Centralized Logging

  • Centralized Logging – Collect all logs into a single system for easy searching and monitoring.
  • Helps with troubleshooting distributed systems and security incident analysis.
